## 一、产品定位与生态关系

### 1.1 BlockCity.vip — 主平台（实勘结果）

通过浏览器实测访问 blockcity.vip，发现这是一个 **城市区块网格游戏**：

```
┌────────────────────────────────────────────────────────────┐
│              BlockCity.vip 页面结构                          │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  BlockCity-广州        来做元宇宙的先行者    [购买区块]       │
│                                                            │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ [A区]  [B区]                  区块网格 (X=01~24)     │  │
│  │                                                      │  │
│  │  0101  0201  0301 ... 1201   1301  1401 ... 2401    │  │
│  │  0102  0202  0302 ... 1202   1302  1402 ... 2402    │  │
│  │  ...                                                 │  │
│  │  0199  0299  0399 ... 1299   1399  1499 ... 2499    │  │
│  │                                                      │  │
│  │  每个方块: 4位编号 + 可选业主名称                       │  │
│  │    例: 1508→"中"  1313→"陈"  1801→"142857"          │  │
│  │    例: 1810→"GTDYGJH吉祥如意平安健康"                 │  │
│  │    例: 0102→"出售"  0508→"转"                       │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                            │
│  [广州]    [圈子]    [市政厅]    [游客]      ← 底部导航     │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**核心玩法**：
- 每个城市有多个区域（A/B/C/D/E/F/G/H/Z），每个区域有数百到数千个方块
- 用户花人民币**购买区块**，购买后可在区块上**命名**（作为广告/自我表达）
- 区块可以**转让/出售**给其他用户
- 底部导航：切换城市、圈子社交、市政厅（治理）、游客模式

**实测城市对比**：
| 城市 | 区块填充率 | 说明 |
|------|-----------|------|
| 广州 | ~60% | 大量区块已被购买和命名 |
| 奥林匹克 | ~0% | 几乎空白，新城市 |

### 1.2 58.tl — 工具站生态

**58.tl** 是 BlockCity 的配套工具站集群，为 BlockCity 用户提供增值服务：

| 子系统 | 域名 | 核心功能 | 与 BlockCity 的关系 |
|--------|------|----------|-------------------|
| 区块城市 | block.58.tl | 区块认领、合并、求购 | 对 blockcity 区块的交易市场 |
| 人气值交易 | bct.58.tl | BCT虚拟货币买卖(3种方式) | 城市人气值的二级市场 |
| NFT头像 | nft.58.tl | 2089个头像的认领/挂售/求购 | 关联 BlockCity 城市的数字收藏品 |
| 购物商城 | mall.58.tl | 用人气值购买商品 | BCT 人气值的消费场景 |
| 互访圈 | hufang.58.tl | 城市间互访社交 | 促进 BlockCity 用户社交互动 |

### 2.3 关键运营发现

1. **BCT交易全部pending** — 31笔订单中0笔实际完成，撮合机制需改善
2. **区块集中A区** — 161个已售区块几乎全是A区(¥1,286)，高价值区域无人问津
3. **用户集中在互访圈** — 390+圈子 vs 31笔BCT交易，社交活跃但交易冷清
4. **存在约40个spam注册** — 随机用户名+奇怪邮箱，需加强注册验证
5. **运营依赖主号** — user_id=1(58tl)持有80+区块、40+圈子，单人贡献突出

## 三、技术架构

```
技术栈:
  后端: PHP 5.6/7.x 原生(无框架)
  前端: jQuery + Bootstrap 3 + Font Awesome + Select2
  数据库: MySQL 5.7 (utf8mb4, PDO)
  服务器: Nginx/Apache + .user.ini

代码规模:
  PHP文件: ~185个
  HTML页面: ~100个
  NFT SVG头像: 2089个
  业务类: 26个(classes/)
  数据库: 30张表 + 3个视图
  API端点: 仅3个
```

### 架构模式

```
每个子模块独立目录，结构一致:
  {module}/
  ├── index.php          主入口
  ├── auth/              登录/注册/登出
  ├── user/              用户功能
  ├── admin/             管理后台
  ├── api/               JSON接口
  └── includes/          模块级header/footer/sidebar

共享层(跨模块):
  config/database.php    数据库连接
  includes/auth.php      认证+CSRF+Session
  includes/functions.php 工具函数
  classes/{23个Class}    业务逻辑层
```

### 安全措施

| 措施 | 实现 |
|------|------|
| 密码 | bcrypt (password_hash) |
| SQL注入 | PDO预处理(全站) |
| XSS | htmlspecialchars() 输出转义 |
| CSRF | 表单token验证 |
| 会话 | HttpOnly + SameSite=Lax + 30min重建ID |
| 记住我 | 32字节随机token，30天滑动过期 |
| 文件上传 | 类型+大小限制(2MB) |

## 五、迭代方向建议

### P0 - 关键问题修复
1. **BCT交易撮合机制** — 31笔订单0完成，考虑自动撮合或引导交易
2. **注册验证** — 40+ spam账号，加验证码/邮箱验证
3. **数据备份规范化** — 定期自动备份

### P1 - 体验优化
4. **移动端适配** — 当前Bootstrap 3，用户手机访问体验差
5. **API接口扩展** — 仅3个API，大量功能需页面刷新
6. **互访圈简化** — 390+圈子但活跃度未知，考虑简化互访流程

### P2 - 功能深化
7. **高价值区块促销** — 当前售出99%是A区，B-H/Z区几乎无人购买
8. **BCT消费场景** — mall商城数据较少，丰富消费场景
9. **NFT与BlockCity联动** — 强化NFT头像与城市区块的绑定
10. **排行榜页面改善** — rankings/下多为静态HTML

### P3 - 架构升级
11. **引入框架** — 消除大量代码重复
12. **前后端分离** — API化 + SPA前端
13. **日志与监控** — 添加运营数据分析
