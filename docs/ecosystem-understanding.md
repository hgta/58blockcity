## BlockCity 核心玩法

### 城市区块网格

```
城市 = 广州为例:
  A区: 列01~12, 行01~199 → 约2,400个区块
  B区: 列13~24, 行01~199 → 约2,400个区块
  C区~H区: 类似结构，更多行
  Z区: 中心区域，价格最高

区块命名举例(来自实勘):
  - 个人品牌: "程", "李府", "陈", "地主", "海藻"
  - 广告: "智能家居", "艺飞钢琴艺术学校", "婚戒特惠"
  - 趣语: "142857", "GTDYGJH吉祥如意平安健康", "不一样的世界"
  - 交易提示: "出售", "转", "转让", "收人气"
```

### 用户行为路径

```
进入 blockcity.vip → 选择城市 → 浏览区块网格
    │
    ├─ 购买区块(花钱) → 命名区块 → 展示个人/商业信息
    │
    ├─ 转让区块 → 其他用户接手
    │
    ├─ 参与圈子 → 与其他城市用户互访 → 提升人气值
    │
    └─ 使用58.tl工具 → BCT交易/NFT头像/商城/互访圈
```

## 人气值(BCT)经济学理解

```
用户行为              贡献的人气值
──────────────────────────────────────────
在 BlockCity 买区块    给所在城市增加人气值
在 BlockCity 命名区块    展示活跃度
互访其他城市            双向增加人气值
持有 NFT 头像          贡献人气权重
参与 BCT 交易           产生流动性

人气值 → BCT余额 → 可在 mall.58.tl 消费
                → 可在 bct.58.tl 交易(¥0.05~0.25/个)
```

## 58.tl 当前的瓶颈与机会

基于数据分析发现的问题：

1. **BCT交易0完成** → 31笔订单全部pending，撮合完全失败
   - 原因推测：用户发布订单后没有自动撮合，纯靠手动联系
   - 建议：实现自动撮合引擎，或引导用户使用platform/mediator模式

2. **区块交易不活跃** → 161个已售区块全是早期交易
   - 区块转让功能可能需要更多曝光

3. **NFT头像使用率低** → 2089个SVG但认领数据少
   - 建议与BlockCity城市绑定，购买区块送NFT

4. **互访圈活跃但转化低** → 390+圈子但缺少商业闭环
   - 互访→人气值→BCT→消费 的链条需要强化

5. **mall商城数据极少** → 几乎无订单/商品
   - BCT消费场景需要真实商品或服务填充
