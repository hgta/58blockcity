# 统一站内信功能

## 问题

当前系统有 `notifications` 表处理系统通知（互访申请、订单状态等），但缺少用户间直接私信功能。刚实现的模特留言也仅限模特页，无法跨场景复用。用户希望在任何看到用户名的地方（商品详情、店铺、互访圈、NFT、模特页等），都能点击头像/用户名给对方发站内信。

## 目标

1. **统一站内信表** `user_messages`：代替 `model_messages`，支持任意用户间私信
2. **可复用发送入口**：全局 `sendMessage(to_user_id)` 函数，任何用户名点击触发
3. **消息收件箱** `/messages/`：查看所有会话，标记已读/未读
4. **未读徽章**：头部显示未读站内信数（与通知徽章分开）
5. **全站集成**：各子站用户名统一可点击

## 范围

| In scope | Out of scope |
|----------|-------------|
| `user_messages` 表 + Message 类 | 实时推送/WebSocket |
| 收件箱页面（会话列表 + 聊天面板） | 图片/文件发送 |
| 模态框快速发送 | 群聊/多人消息 |
| 未读计数 + 头部徽章 | 消息撤回/编辑 |
| 替换 model_messages 迁移数据 | 语音/视频消息 |
| mall/block/hufang/nft/bct 全子站集成 | |

## 为什么是独立表而非复用 notifications

`notifications` 表设计为系统事件通知（类型枚举 `visit_request`/`order_paid` 等），与用户间私信语义不同。私信需要会话分组、双向查看、已读标记，独立表更清晰。已有的 `model_messages` 将在实现时迁移合并。
