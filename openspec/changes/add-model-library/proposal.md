# 增加模特库功能

## 问题

目前商城商品详情页仅展示店铺信息，无法体现"谁拍了这个商品"。对于服装、饰品、生活类商品，"真人展示"是重要的购买决策参考因素。需要一个独立的模特库系统：

- 后台管理模特档案（昵称、站内用户、性别、年龄、社交账号、身材数据等）
- 商品编辑时可关联模特
- 商品详情页展示模特昵称，点击进入模特专属展示页
- 模特页聚合该模特拍摄的所有商品及图集
- 排行榜增加模特维度排行

## 目标

1. **后台模特管理**：CRUD 模特档案，关联站内用户
2. **商品-模特关联**：商品编辑页可选模特（1:1），商品详情页展示
3. **模特专属页面**：展示基本信息 + 关联商品 + 图片聚合 + 点赞
4. **模特排行榜**：关联商品数、点赞数、评论数排行
5. **SEO**：模特页标题/描述/结构化数据

## 范围

**In scope**:
- `models` 表 + `model_likes` 表
- `Model` 类（CRUD、点赞、排行查询）
- Admin 模特管理页
- 商品编辑页增加模特选择
- 商品详情页展示模特信息
- 模特详情页
- 模特排行榜（3 维度）
- 模特页 SEO 元数据

**Out of scope**:
- 模特独立照片上传（图集自动聚合关联商品 images）
- 模特申请/审核流程
- 模特私信/预约功能
- 前台用户自助申请成为模特
