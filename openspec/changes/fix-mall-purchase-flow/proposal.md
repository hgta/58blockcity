# 修复商城购买主流程

## 背景

58人气值商城已完成基础的浏览、购物车、结算、支付功能，但通过对代码的详细审查，发现购买主线存在 2 个 Bug 和 1 个设计缺陷。

## 问题

### Bug 1："立即购买"流程断裂 (P0)

`product/detail.php` 第536行：
```html
<a href="../cart/checkout.php?product_id=123&quantity=1">立即购买</a>
```

`checkout.php` 第34行只读购物车，完全忽略 GET 参数：
```php
$cartItems = $cart->getCartItems($userId);
```

**后果**：用户点"立即购买"时，结算的是购物车里已有全部商品，不是当前浏览的商品。

### Bug 2：取消订单不恢复库存 (P0)

`cancel_order.php` 调用 `Order::cancelOrder()` 只改订单状态，没有恢复被锁定的商品库存。

**后果**：用户取消订单后，被扣减的库存永久丢失，导致可售库存越来越少。

### 缺陷：BCT 支付无凭证记录 (P1)

支付确认页 `user/pay.php` 收到"我已支付"后直接改 `orders.status = 'paid'`：
- 不调用 `BCTTransaction::create()` 创建交易凭证
- 不记录交易号、金额、城市等对账信息
- 无任何验证链路

**后果**：一旦产生纠纷（买家说付了、卖家说没收到），无任何数据可供对账。

## 目标

修复 2 个 Bug + 1 个设计缺陷，保障购买主流程完整闭环：
- "立即购买"能正确传导当前商品到结算页
- 取消订单后商品库存正确恢复
- BCT 支付时创建交易凭证记录

## 范围

### 本次包含

- 修复 `product/detail.php` 的"立即购买"按钮逻辑
- 修复 `cancel_order.php`，取消时恢复库存
- 修改 `user/pay.php`，支付确认时创建 BCTTransaction 记录

### 本次不包含

- 运费/优惠逻辑
- BCT 链上自动验证
- 前端 UI 重构
- 订单进度可视化

## 成功指标

- "立即购买"正确将当前商品加入购物车并跳转结算
- 取消订单后商品 `stock` 字段恢复
- 每次支付确认后 `bct_transactions` 表有一条对应记录
