# 重新设计 NFT 售卖页功能和布局样式

## 背景

当前 `nft/nft/sale_list.php` 是 NFT 销售市场的主页，但存在多项数据和设计问题，导致页面信息不完整、筛选器失效、展示效果差。

## 问题

### 数据层断裂

`getSaleList()` SQL 只 JOIN 了 `nft_transactions` 和 `users`，但前端模板引用了未查出的字段：
- `$sale['rarity']` — 稀有度永远是空，badge 无意义
- `$sale['city']` — 城市名永远是空，无法获知头像归属城市
- 城市筛选下拉的 SQL 条件被注释掉，选了不生效

### 布局设计问题

- SVG 头像被 `rounded-circle` 强制裁成圆形
- Bootstrap 通用卡片缺乏市场感，信息密度低
- 只展示编号+价格+卖家，缺少求购热度、城市归属等关键信息
- 分页用 Bootstrap 默认样式，与全站风格不统一

## 目标

重新设计售卖页，修复数据问题、美化布局、增强信息展示，打造有吸引力的 NFT 交易市场入口。

## 范围

### 本次包含

- 修复 `getSaleList()` 和 `getTotalSaleCount()` SQL：JOIN cities 取城市名，启用 city 筛选
- 新增求购数统计子查询，每个卡片展示"🔥N人求购"
- 删除失效的 rarity（稀有度）相关筛选和展示
- 全面重写 `sale_list.php` 布局和样式：
  - 顶部统计摘要栏（在售数、覆盖城市数）
  - 水平筛选栏（编号搜索、城市下拉、货币、排序）
  - 自适应卡片网格，圆形头像 + 橙色描边
  - 卡片信息层级：编号 → 城市 → 价格 → 卖家+求购热度
  - 每张卡片整卡可点击跳详情页
  - 带省略号的现代化分页

### 本次不包含

- 详情页 `view.php` 修改
- 购买/交易流程改动
- 支付相关改动
- 后端 API 接口新增

## 成功指标

- 城市筛选下拉正常工作，选城市后仅显示该城市挂售的 NFT
- 每张卡片正确展示"📍 北京"等城市归属信息
- 有求购记录的 NFT 显示"🔥N人求购"
- 页面视觉上有明显的市场感和吸引力
