# Proposal: 全站移动端响应式优化

## 问题

网站（mall.58.tl / v.58.tl）在手机端访问时存在多个体验问题：

| 问题 | 影响范围 | 严重度 |
|------|---------|--------|
| **共享头部导航缺少汉堡菜单** — 7个导航链接在768px以下仅缩小字号+换行，375px屏上极其拥挤、难以点击 | 全站所有页面 | 🔴 严重 |
| **支付设置页无响应式** — 表格（城市/区块/启用）在手机上溢出 | 店主核心操作 | 🟡 中等 |
| **商品管理工具栏无适配** — 筛选tabs+搜索+批量操作在手机堆叠混乱 | 店主核心操作 | 🟡 中等 |
| **多个表单页无@media** — 开店、个人资料、安全设置等页在手机未优化 | 用户设置 | 🟡 中等 |

## 目标

让手机用户在所有核心页面上都能顺畅浏览和操作，无卡点。

## 范围

**包含：**
- `shared/header.php` — 添加汉堡菜单
- `mall/shop/payment-settings.php` — 表格移动端优化
- `mall/shop/products.php` — 工具栏移动端适配
- `mall/shop/create.php` — 表单响应式
- `mall/user/profile.php` — 表单响应式

**不包含：**
- hufang/ 模块（已有系统化 CSS 框架，34 个断点）
- admin/ 模块（已有统一 admin.css，侧边栏+汉堡菜单+表格横向滚动）
- city/ 静态页面（独立维护）

## 成功标准

1. 手机（375px-414px）访问时，头部显示汉堡图标，展开后显示完整导航菜单
2. 支付设置表格在小屏可正常操作（横向滚动或卡片化）
3. 商品管理工具栏在小屏合理堆叠
4. 所有表单页在 375px 屏上输入框不溢出，按钮触控区域 > 44px
