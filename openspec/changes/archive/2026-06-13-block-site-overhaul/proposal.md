# Block 子站点系统性改造提案

## 背景与问题

Block 子站点（https://block.58.tl）是 58 区块城市的核心交易入口，承载城市区块的浏览、认领、售卖与求购功能。当前站点存在以下影响用户体验与平台信任度的关键问题：

1. **首页显示异常**：`shared/header.php` 与 `block/index.php` 各输出一条城市定位条，导致同时显示两行定位信息（如"佛山"与"未知城市"），且字母导航完全无 CSS 样式，呈现浏览器默认链接堆叠状态。
2. **性能瓶颈**：`city.php` 全景模式一次性生成约 90,000 个 DOM 节点（9 区 × 99×101 个 `<span>`），低端设备直接卡死，无 loading 状态，白屏时间长。
3. **交易功能不完整**：
   - `sale_list.php` 语义矛盾——注释写"for sale"，SQL 却查 `status='sold'`（已售），用户无法看到真正在售的区块。
   - `purchase_list.php` 只有展示，无"发布求购"入口。
   - 各列表页无分页（`claim_list` 硬编码 30 条，`purchase_list` 硬编码 50 条）。
4. **安全与数据问题**：`sale_list.php` 的 LIMIT 采用字符串拼接；Z 区价格计算写死 1000 元，与配置的复杂规则不匹配。
5. **视觉与一致性缺失**：三列表页（sale/claim/purchase）各自内嵌 `<style>`，风格不统一；移动端 99×101 地图 cell 缩至 20px，几乎无法点击；区块拥有者只显示 `user_id` 数字。
6. **数据空洞**：首页"元宇宙特色"纯文案，无实时平台数据支撑，缺乏信任感。

## 目标

将 block.58.tl 从一个"功能可用"的初级站点，升级为**高性能、功能闭环、数据驱动、视觉统一**的专业级区块交易平台。

## 范围

### 纳入范围
- `block/index.php` — 首页修复 + 实时数据面板 + 城市搜索
- `block/city.php` — 全景模式性能重构 + 单区交互优化
- `block/sale_list.php` — 语义修正 + 分页 + 筛选
- `block/claim_list.php` — 分页 + 筛选
- `block/purchase_list.php` — 分页 + 筛选 + 发布求购入口
- `block/top200city.php` — 补充实际内容（热门城市排行）
- `shared/header.php` / `shared/footer.php` — 若涉及 block 站的公共样式冲突
- `assets/css/main.css` — 补充 block 站所需通用样式（字母导航、列表卡片等）
- `classes/Block.php`、`classes/City.php` — 按需新增查询方法

### 不纳入范围
- 后端区块所有权链上存证逻辑（超出当前改造范围）
- BCT 支付接口改造（由 BCT 子站独立演进）
- 管理后台功能
- 多语言国际化

## 成功标准

| 指标 | 当前状态 | 目标状态 |
|------|---------|---------|
| 首页定位条 | 显示两行，冲突 | 仅一行，由 header 统一控制 |
| 字母导航 | 无样式，堆叠 | 居中胶囊式导航，hover/active 态 |
| 全景模式首屏时间 | 低端机卡死/白屏 3s+ | < 1s，有 loading 动画 |
| 全景模式 DOM 节点 | ~90,000 个 span | < 100 个节点（canvas/svg） |
| 列表页分页 | 无 | 统一 20 条/页，页码导航 |
| 列表页筛选 | 无 | 支持按城市、区域、价格筛选 |
| 售卖语义 | 查询 sold（已售） | 查询 for_sale / listed（在售） |
| 发布求购 | 无入口 | 顶部有"发布求购"按钮，跳转表单页 |
| 首页数据面板 | 纯文案 | 展示实时统计：城市数、激活区块、用户数 |
| 移动端地图可点击 | cell 20px 难以命中 | cell 最小 40px，或切换为列表模式 |

## 风险与假设

- **假设**：数据库 `blocks` 表存在可用于区分"已售"和"在售"的字段（如 `is_listed`、`list_price`），若不存在需在实施阶段新增迁移。
- **风险**：全景模式改为 canvas 后，原有 `title` 悬停提示需用 JS 重新实现；合并区块高亮逻辑需同步移植。
- **风险**：`city/city.js`（Amap IP 定位）在不同浏览器/网络环境下的兼容性已有一次回滚历史，本次改动不触碰该文件。
