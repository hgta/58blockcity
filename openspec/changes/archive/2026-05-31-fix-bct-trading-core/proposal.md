# 修复 BCT 交易核心系统

## 背景

BCT（人气值）交易是 58.tl 虚拟经济系统的核心引擎。用户通过买卖不同城市的 BCT 实现价值交换，同时也是商城消费的支付基础。

## 问题

通过对代码和线上数据的深度分析，发现 BCT 交易系统**当前完全不可用**：

```
线上数据: 31 笔订单，0 笔完成，完成率 0%
代码分析: 3 个致命 Bug + 3 个设计缺陷
```

### 致命 Bug

1. **`BCTTransaction` 类不存在**：`BCTOrder::executeTrade()` 中 `new BCTTransaction($this->pdo)` 会直接抛错，导致所有自动撮合失败
2. **前端"确认交易"是假的**：`bct/index.php` 第678行的 `executeTrade()` 只是 `setTimeout + alert`，没有后端 API
3. **两条创建路径不一致**：`trade.php`（简化版，不验证余额）vs `process_order.php`（完整版，验证余额+冻结），写入同一张表但行为不同

### 设计缺陷

4. **自动匹配无价格条件**：买单 ¥0.05 和 ¥0.25 被同等对待，只要城市相同、方向相反就匹配
5. **部分成交不调整冻结余额**：100 BCT 成交了 30，仍冻结全部 100
6. **价格调整算法未启用**：`autoAdjustPrice()` 定义后从未被调用，无定时触发

## 目标

修复所有致命 Bug 和设计缺陷，使 BCT 交易系统具备**基本的交易闭环能力**：
- 用户能发布买卖订单
- 平台能自动撮合匹配的订单
- 用户能手动确认成交
- BCT 余额和冻结正确流转
- 价格随供需动态调整

## 范围

### 本次包含

- 创建缺失的 `BCTTransaction` 类
- 修复 `index.php` 成交逻辑（前端连后端 API）
- 统一订单创建路径（废弃 `trade.php` 简化版，只保留完整流程）
- 匹增加价格条件（买卖价格交叉即可成交）
- 修复部分成交的冻结余额
- 新增定时任务脚本（cron job）自动撮合 & 调价

### 本次不包含

- 商城 BCT 支付接入（下一期）
- 前端 UI 改版
- BCT 经济模型大改
- 互访圈与 BCT 联动

## 成功指标

- 系统能完成至少一笔端到端的 BCT 交易
- 自动撮合能正确匹配买卖单
- 定时任务每 N 分钟运行一次，处理 pending 订单
- 价格随供需变化自动调整
