# 统一后台管理系统 (Unified Admin Dashboard)

## 问题陈述

当前 58 区块城市各子站点的后台系统存在严重的碎片化问题：

1. **样式不统一**：mall 和 nft 后台各自内嵌 400+ 行几乎相同的 CSS；hufang 使用 Bootstrap + 自定义类；block/bct 使用 Bootstrap 默认样式。视觉体验割裂，维护成本极高。
2. **无跨站点导航**：管理员在 mall 后台时无法直接跳转到 nft 后台，必须手动改 URL，管理效率低下。
3. **权限检查分散**：各子站自行判断管理员身份（有的 `user_id==1`，有的 `role=='admin'`），缺乏统一标准。
4. **缺少总控后台**：主站没有统一的管理后台来汇总展示所有子站点的核心数据（用户数、交易量、活跃度等）。
5. **block 子站缺少管理后台**：block 仅有用户中心，没有管理员专用的区块/城市/交易管理后台。

## 目标

构建一个**统一的现代化后台管理框架**，实现：

- 主站总控后台：汇总所有子站点核心数据，提供全局视图
- 各子站专属后台：保留业务特色，统一视觉和交互
- 共享样式框架：一套 CSS + 组件，所有后台共用
- 跨站点导航：任意后台均可一键切换至其他子站后台
- 统一权限管理：基于角色的访问控制（RBAC）

## 范围

### In Scope

| 模块 | 内容 |
|------|------|
| 共享框架 | `shared/admin/` 统一头部/尾部/侧边栏/菜单配置 |
| 共享样式 | `assets/css/admin.css` 现代化深色主题管理样式 |
| 主站总控后台 | `admin/dashboard.php` 汇总数据看板 |
| block 管理后台 | `block/admin/` 区块/城市/交易管理 |
| 现有后台迁移 | mall/admin/、nft/admin/、hufang/admin/ 逐步接入统一框架 |
| 权限统一 | `classes/AdminAuth.php` 统一 RBAC 权限检查 |
| 跨站导航 | 顶部全局导航栏，支持所有子站后台间切换 |

### Out of Scope

- 前台页面样式改造（已有 `block-site-overhaul` 变更覆盖）
- 数据库结构大改（仅新增 `admin_roles` / `admin_permissions` 表）
- 移动端后台适配（后台以桌面端为主，基础响应式即可）

## 影响范围

- **新增文件**：~15 个（共享组件、总控后台、block 管理后台、权限类）
- **修改文件**：~10 个（现有 mall/nft/hufang 后台接入统一框架）
- **零风险降级**：各子站原有后台保留备用，可随时回退

## 成功标准

1. 所有后台页面共用同一套 `admin.css`，不再有任何内联 CSS > 50 行
2. 管理员在任意后台均可通过顶部导航切换至其他子站后台
3. 主站总控后台能展示至少 5 个核心汇总指标（总用户、总交易、总区块、今日活跃等）
4. block 管理后台能完成用户区块管理、城市管理和交易审核
5. 统一权限检查覆盖所有后台入口，拒绝非管理员访问
