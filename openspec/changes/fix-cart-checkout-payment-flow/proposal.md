# 购物车、结算与支付流程修复改造

## 问题陈述

当前商城的购物车、结算、支付全流程存在多个影响用户体验和交易安全的严重问题：

1. **价格符号严重混乱**：BCT 人气值价格在购物车、结算页、订单详情、支付确认页中全部被错误显示为 `¥` 人民币符号，极易误导用户。
2. **多店铺结算缺陷**：结算页硬编码只取第一件商品的 `shop_id`，购物车中若包含多个店铺的商品，订单信息会出错。
3. **支付体验极差**：用户需手动前往 blockcity.vip 转账，再回到页面点击"我已支付"，没有任何支付状态验证或防欺诈机制。
4. **库存缺少超时释放**：下单时直接扣减库存，若用户不支付，库存将永久丢失，无超时自动取消机制。
5. **订单字段不一致**：`checkout.php` 调用 `addOrderDetail()`，但 `Order.php` 中存在 `addOrderDetail()` 和 `addOrderItem()` 两个方法，字段映射不一致。
6. **缺少物流信息**：卖家发货仅修改订单状态，无法填写快递公司和运单号。

## 目标

修复并优化购物流程，实现：

- 所有 BCT 价格统一显示 `Ⓟ` 人气值符号，人民币价格显示 `¥`
- 购物车按店铺分组，支持多店铺分别结算（或拆分为子订单）
- 支付确认页增加支付倒计时，超时自动取消订单并释放库存
- 统一订单添加方法，消除字段不一致
- 卖家发货时支持填写物流公司和运单号

## 范围

### In Scope

| 模块 | 内容 |
|------|------|
| 价格符号修复 | `mall/cart/index.php`、`checkout.php`、`user/order_detail.php`、`user/pay.php` 统一 BCT 符号 |
| 购物车增强 | 按店铺分组展示，增加"选中"功能，支持批量结算 |
| 结算页修复 | 修复多店铺硬编码问题，每个店铺生成独立订单 |
| 支付流程 | 增加支付倒计时、超时自动取消、库存回滚 |
| 订单字段统一 | 统一 `addOrderDetail` 方法，确保字段一致 |
| 物流信息 | `orders` 表增加物流字段，卖家发货页面支持填写 |

### Out of Scope

- 接入真正的 BCT 链上支付 API（当前仍为手动转账确认模式）
- 前台商品列表/详情页的价格显示（已在之前变更中修复）
- 退款流程自动化
- 优惠券/满减系统

## 影响范围

- **修改文件**：`mall/cart/index.php`、`mall/cart/checkout.php`、`mall/user/pay.php`、`mall/user/order_detail.php`、`mall/shop/orders.php`、`classes/Order.php`、`classes/Cart.php`
- **新增文件**：`mall/user/orders.php` 物流提交接口（如需要）
- **数据库变更**：`orders` 表增加 `expire_at`、`shipping_company`、`tracking_no` 字段

## 成功标准

1. 购物车、结算页、订单详情、支付页中所有 BCT 价格正确显示 `Ⓟ` 符号，无 `¥` 混淆
2. 购物车包含多个店铺商品时，结算生成对应数量的独立订单
3. 待付款订单超过 30 分钟未支付自动变为 cancelled 并释放库存
4. 卖家在后台发货时可填写物流公司和运单号，买家可在订单详情查看
5. `Order::addOrderDetail` 方法被唯一使用，无字段映射错误
